__ТТ-1: "Уверенный пользователь"__\
__Цель:__ Проверить сценарий активного пользователя, который быстро выполняет все действия\
__Охват:__ Сайт "Школа 21" раздел "Как мы учим"; Страница аппликанта раздел "Авторизация"; Страница аппликанта раздел "Обучение"\
__Критерии успеха:__ Переход между страницами осуществляется без задержек(< 3 сек) и ошибок \
__Начальная точка:__ Пользователь находиться на странице https://21-school.ru/methodology\
__Действия:__
1. Открыть страницу https://21-school.ru/methodology 
2. Быстро нажать кнопку "Вход" (происходит переход на сайт аппликанта) 
3. Ввести валидные логин и пароль, нажать кнопку "Вход" 
4. Закрыть страницу аппликанта, снова ввести адрес https://21-school.ru/methodology и нажать кнопку "Вход" 

__Итог:__ Все функции сайта загружались быстро и сохранили данные для повторого входа на страницу аппликанта

__ТТ-2: "Антиобщественный тур"__\
__Цель:__ Проверить сценарий пользователя, который переходит по всем возможным ссылкам и отправляет заявки на несуществующие адреса\
__Охват:__ Сайт "Школа 21" раздел "Как мы учим" https://21-school.ru/methodology\
__Критерии успеха:__ Открывая все разделы подряд, сайт открывает последний нажатый раздел. При попытке отправить письмо на несуществующий электронный адрес или телефон, сайт не пропускает заявку \
__Начальная точка:__ Пользователь находиться на странице https://21-school.ru/methodology \
__Действия:__ 
1. Переключаться по всем возможным разделам сайта
2. Попытка ввести в любую из форм заявки невалидные почту или телефон (@nail.com @ggg.com, @. и т.д.)
3. Попытки отправить пустую (или с пробелами) заявку 
4. Попытки заполнять форму заявки, возвращаться назад и повторять дейтсвие
5. Попытки многократно нажимать по разным интеркативным элементам
6. Попытки скролить (быстро перематывать) страницу
7. Сделать все те же действия в разных браузерах 

__Итог:__ Все функции обрабатываются согласно функционалу сайта, без выведения ошибок. При введении невалидных данных в формы заявки, она не отправляется, выдавая рекомендации


__ТТ-3: "Неуверенный пользователь"__\
__Цель:__ Проверить сценарий пользователя, который начинает делать действие и тут же его прекращает\
__Охват:__ Сайт "Школа 21" раздел "Как мы учим" https://21-school.ru/methodology \
__Критерии успеха:__ Открывая, а затем отменяя действие, сайт не выдаст ошибки, вернется в исходное состояние. Заполненные формы заявки после выхода из них сохранятся. \
__Начальная точка:__ Пользователь находиться на странице https://21-school.ru/methodology \
__Действия:__
1. Начать переход по любым интерактивным кнопкам (Логотип; О "Школе 21"; Корпоративное обучение и т.д.), после нажатия, сразу возвращать страницу в исходное состояние кнопкой браузера "отмена"
2. Начать заполнять форму заявки (поступление; открыть кампус и т.д.), после заполнения вернуть страницу в исходное состояние кнопкой браузера "отмена"
3. Открыть любое видео, посмотреть его и вернуть страницу в исходное состояние кнопкой браузера "отмена"

__Итог:__ После нажатия кнопки отмены, сайт возвращается в исходное состояние, не выдает ошибок. Формы заявки сбрасываются, а видео не воспроизводиться с того момента, на котором просмотр был прерван после возврата
