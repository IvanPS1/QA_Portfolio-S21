## Task 3. Базы данных

*   Напиши отличия реляционных БД от нереляционных, а также плюсы и минусы каждого вида.
*   Напиши отличия СУБД между собой (например, чем отличается MySQL от PostgreSQL).

## Task 3. Решение:

**1. Отличия реляционных БД от нереляционных:** 

|  | Реляционные БД (SQL) | Нереляционные БД (NoSQL) |
| - | - | - |
| Отличия | Хранят данные в виде таблиц с фиксированными столбцами и строками, используя язык SQL для запросов<br> Обеспечивают высокую целостность данных и поддержку транзакций, что важно для систем, где требуется высокая надежность | Предлагают более гибкие модели хранения, такие как графы, документы или ключ-значение<br> Не всегда обеспечивают строгую целостность данных и поддержку транзакций, как реляционные СУБД |
| Плюсы | Высокая целостность данных<br> Надежность<br> Хорошо подходит для сложных запросов и транзакций | Высокая гибкость<br> Маштабируемость <br> Хорошо подходит для больших объёмов и неструктурированных данных |
| Минусы | Менее гибкие в отношении структуры данных<br> Масштабирование может быть сложнее (особенно для больших объемов данных) | Меньшая целостность по сравнению с SQL <br> Сложность в реализации сложных запросов |


**2. Отличия СУБД между собой:**

**Реляционные СУБД:**

* Организация данных в виде таблиц с четкими связями между ними
* Используют язык SQL для запросов
* Примеры: MySQL, PostgreSQL, Oracle Database, Microsoft SQL Server
* Подходят для структурированных данных и транзакционных систем

**Колоночные СУБД:**

* Хранят данные по столбцам, а не по строкам
* Оптимизированы для аналитических запросов и обработки больших объемов данных
* Примеры: ClickHouse, Vertica, Google BigTable

**Графовые СУБД:**

* Представляют данные в виде графов, где вершины – объекты, а ребра – связи между ними
* Подходят для анализа связей и зависимостей между данными
* Примеры: Neo4j, Amazon Neptune

**СУБД «ключ-значение»:**

* Простейшая модель, хранящая данные в виде пар "ключ-значение"
* Высокая скорость доступа к данным, часто используются для кэширования
* Примеры: Redis, Memcached

**NoSQL СУБД:**

* Предлагают различные модели данных: документоориентированные, графовые, ключ-значение и колоночные
* Не используют SQL, часто имеют собственный язык запросов или API
* Примеры: MongoDB, Redis (ключ-значение), Neo4j (графовая) 
* Подходят для больших объемов неструктурированных данных, масштабируемости и гибкости



| Характеристика    | PostgreSQL                  | MySQL                      |
|-|-|-|
| **Типы данных**   | Поддержка JSON, геоданных  | Обычные SQL-типы           |
| **Производительность** | Оптимизирован для сложных запросов | Быстрее на простых операциях |
| **Транзакции**    | Полная поддержка ACID      | Зависит от движка (InnoDB) |
| **Лицензия**      | Open-source (свободный)    | Oracle (есть платные версии) |
