## Task 2. Причины появления дефектов

Для каждой из нижеперечисленных причин появления дефектов напиши своё видение, как можно было бы его избежать/предотвратить.

1. ошибки в спецификации / техническом задании;
2. ошибки при написании кода разработчиком (например, из-за слишком сложной логики функциональности, недостаточности времени для написания кода и т. д.);
3. плохая коммуникация в команде;
4. ошибки в инструментах разработки.

## Task 2. Решение

1. Более детальный разбор ТЗ (ревью с командой)
2. Разбить сложный функционал на более мелкие модули, чтобы легче писать код. Но тогда может возникнуть дефект на уровне совместимости модулей (просмотр кода другими коллегами)
3. Устраивать совместное времяпрепровождение; совместные игры вне работы; создание более приятной атмосферы для лучшей коммуникации сотрудников (игровые комнаты, в которых можно провести время на перерыве; спорт-площадка); использовать инструмены коллаборации
4. Использовать более передовые инструменты
