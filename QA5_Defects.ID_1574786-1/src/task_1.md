## Task 1. Решение

### 1. Функциональные дефекты

* Эти дефекты связаны с функциональностью конкретного программного компонента. Проще говоря, любой компонент в приложении или на сайте, который работает не так, как задумано, является функциональной ошибкой. (Пример: кнопка login не позволяет пользователям войти в систему)

### 2. Логические дефекты 

* Это дефекты, которые могут привести к неожиданному поведению программного обеспечения и внезапным сбоям. (Пример: вместо сложения двух числе они деляться друг на друга)

### 3. Дефекты рабочего процесса

* Такие дефекты связаны с пользовательской навигацией по программному продукту. (Пример: пользователь ввел свои данные и нажал кнопку "Сохранить и выйти" но программа завершила работу не сохранив данные)

### 4. Дефект на уровне модулей

* Это дефекты, которые возникают на этапе проверки отдельных модулей. Их разработчик чаще всего находит сам при покрытии автотестами.

### 5. Системные дефекты

* Системные дефекты возникают в основном тогда, когда две или более части кода не могут взаимодействовать друг с другом из-за их несовместимости.

### 6. UI дефекты

* Проблемы с отображением элементов интерфейса, верстка, навигация. (Пример: некорректно отображается логотип компании) 

### 7. Дефекты производительности

* Проблемы с быстродействием, зависания, ошибки, связанные с нагрузкой. (Пример: страница загружается дольше 5 секунд при стабильном интернете)

### 8. Дефекты безопасности

* Уязвимости, позволяющие несанкционированный доступ к данным или системе. (Пример: SQL/XSS инъекции)

### 9. Дефекты совместимости

* Проблемы взаимодействия с другими компонентами системы или различными платформами. (Пример: приложение доставки не взаимодействует со сторонней платформой бухаглтерии)

### 10. Дефекты установки и конфигурации

* Проблемы при установке или настройке программного обеспечения. 

### 11. Дефекты документации

* Ошибки в пользовательской или технической документации. 

### 12. Регрессионные дефекты

* Ошибки, которые возникают в ранее рабочей функциональности после внесения изменений в код. (Пример: после обновления перестала работать кнопка "login", хотя до обновления она работала)

### 13. Локализационные дефекты

* Проблемы с переводом или отображением интерфейса на разных языках. (Пример: плейсхолдер при переводе на немецкий язык не помещается в поле)

### 14. API дефекты 

* Это дефекты которые возникают при несоответствии API - ответов документации (Пример: API возвращает 500 ошибку при валидном запросе)

### 15. Дефекты данных

* Дефекты возникающие при некорректном сохранении данных

### 16. Дефекты алгоритмов 

* Ошибки в математических или бизнес-алгоритмах. (Пример: Неправильный расчет скидки при сумме заказа от 5000 руб.)
