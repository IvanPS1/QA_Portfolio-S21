## Task 4. Решение:

#### 1. Схема работы HTTPS-протокола:

Пример: пользователь хочет перейти на сайт Тест.ру, который работает по безопасному протоколу HTTPS.

1. Браузер пользователя просит сайт предоставить SSL-сертификат.

2. Сайт на HTTPS отправляет сертификат.

3. Браузер проверяет подлинность сертификата в центре сертификации.

4. Браузер и сайт договариваются о симметричном ключе при помощи асимметричного шифрования.

5. Браузер и сайт передают зашифрованную информацию.
    


    
#### 2. Основные коды состояния HTTP делятся на пять групп, каждая из которых представляет определенный тип ответа сервера. 

Подробное описание групп кодов состояния HTTP:
* 1xx (Информационные):\
Эти коды указывают на то, что запрос был получен и продолжается обработка. Они не являются ошибками и обычно не отображаются пользователю.

* 2xx (Успешные):\
Эти коды сигнализируют об успешном завершении запроса. Наиболее распространенный код в этой группе - 200 OK, означающий, что запрос был успешно обработан.

* 3xx (Перенаправления):\
Эта группа кодов указывает, что для завершения запроса требуется выполнить дополнительные действия, такие как переход по другому URL.

* 4xx (Ошибки клиента):\
Коды этой группы сообщают об ошибках со стороны клиента, то есть, запрос содержит неверные данные или не может быть выполнен по какой-либо причине, например, 404 Not Found (ресурс не найден).

* 5xx (Ошибки сервера):\
Эти коды указывают на ошибки со стороны сервера, то есть, сервер не может обработать запрос из-за проблем на своей стороне, например, 500 Internal Server Error (внутренняя ошибка сервера).
