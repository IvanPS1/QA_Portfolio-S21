## Task 1. Клиент-серверная архитектура
1. Изучи трёхзвенную и многозвенную клиент-серверную архитектуру. Выпиши основные компоненты многозвенной клиент-серверной архитектуры.


2. Напиши назначение каждого из выписанных компонентов многозвенной архитектуры.

3. Изучи вопрос отказоустойчивости систем на основе «горячего» и «холодного» резерва. Напиши определения, основные характеристики (готовность, синхронизация данных, время переключения, стоимость), плюсы и минусы для каждого вида.

## Task 1. Решение

### 1. 
* __Клиенты__: это устройства или приложения, которые запрашивают информацию или услуги у сервера. Клиенты могут быть как программными приложениями (например, веб-браузеры, мобильные приложения), так и аппаратными устройствами (например, смартфоны, планшеты, терминалы, так называемые «тонкие клиенты»).

* __Серверы__: это компьютеры, которые предоставляют запрашиваемую информацию или услуги клиентам. Серверы могут выполнять различные функции, такие как хранение данных, обработка запросов, вычисления и т. д.

* __Протоколы обмена данными__: это правила или наборы инструкций, которые определяют, как клиенты и серверы обмениваются информацией. Некоторые из наиболее распространенных протоколов: HTTP/HTTPS (гипертекстовые протоколы), стек протоколов TCP/IP (набор правил, описывающих, как компьютеры соединяются и передают информацию друг другу), протоколы отправки и получения почты (SMTP, POP3, IMAP).

* __Базы данных__: это хранилища информации, которые используются на серверной стороне для хранения и управления данными. Базы данных позволяют серверу эффективно хранить, организовывать и извлекать информацию по запросу клиента.

* __Сеть__: это инфраструктура, которая обеспечивает связь между клиентами и серверами. Упрощенно говоря, сети могут быть локальными (LAN) и глобальными (WAN). 

* __Система безопасности__: это компонент, который обеспечивает защиту данных, передаваемых между клиентами и серверами. Этот компонент может включать в себя шифрование данных, аутентификацию и авторизацию клиентов, защиту от несанкционированного доступа и другие меры безопасности.

* __Хранение и обработка данных__: это компоненты, связанные с хранением данных на серверах и их обработкой. Это может включать в себя серверные операционные системы, системы управления базами данных (СУБД), серверы приложений и другие компоненты, необходимые для эффективной работы клиент-серверной архитектуры.

### 2. 
* __Клиент__ - приложение инициирующее запрос

* __Сервер__ - мощный компьютер который получает информацию от Клиента и обрабатывает бизнес логику

* __Протоколы обмена данными__ - правила обмена данными

* __База данных__ - хранение и управление данными

* __Сеть__ обеспечивает передачу данных между клиентами и серверами по протоколам обмена данными

* __Система безопасности__ обеспечивает безопасность соединения

### 3. 

* __Горячий резерв__ - резервная система работает параллельно с основной, постоянно синхронизируется и готова мгновенно взять на себя нагрузку при отказе

* __Холодный резерв__ - резервная система отключена и активируется только после отказа основной

* __Готовность__ - способность системы немедленно заменить основную систему

* __Синхронизация данных__ - актуальность данных в резервной системе относительно основной

* __Время переключения__ - интервал между отказом основной системы и полным переходом на резервную

* __Стоимость__ - затраты на развертывание и поддержку резерва

| Критерий | Горячий резер | Холодный резерв |
| :- | - | - |
| Готовность | Мгновенно | Требуется время |
| Синхронизация данных | Актуализация в реальном времени | Данные могут быть устаревшими |
| Время переключения | миллисекунды | от нескольких минут до нескольких часов |
| Стоимость | Высокая | Низкая |
| Плюсы | Мгновенное переключение <br> Полная синхронизаия данных <br> Автоматизация| Низкая стоимость обслуживания <br> Простота развертывания |
| Минусы | Дорогое одслуживание <br> Сложность настройки <br> Ибыточность | Потеря данных при ЧС <br> Долгое восстановление |
